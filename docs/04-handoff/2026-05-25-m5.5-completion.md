### 关键设计点

#### 1. RBAC 双重隐藏

- **第一道**：`features/shell/app_shell.dart` 的 sidebar `if (me?.role == 'admin')` 才渲染
  "管理后台" 入口；普通 user 树里压根没这个按钮。
- **第二道（兜底）**：`core/router.dart` redirect 加 `_adminRoutes` 集合，loggedIn 且
  `value.me.role != 'admin'` 且 matchedLocation ∈ `_adminRoutes` → 弹回 `/chat`。
  即便用户直接在地址栏敲 `/admin`、或前端入口被绕过，也回 chat。
- **第三道（更兜底）**：后端 `/admin/*` 路由全部走 `Depends(require_role("admin"))`，
  非 admin 401/403。`admin_api_test.dart 403 case` 验证 client 把它冒泡成 DioException
  让 UI 显示 "加载失败"。
- AdminDashboard 内 build 顶部再自检 `me.role != 'admin' → 显示 admin_forbidden`，
  防止有人改 router redirect 漏掉拦截，仍能在页面看到友好提示而不是崩。

#### 2. 任务面板轮询策略

- 进入面板首次拉一次，之后只在"列表里至少一项 status 为 `queued` 或 `running`" 时
  调度 `Timer(3s, _refresh)`；任务都 done/failed 后停止轮询，节省请求。
- 切换 status filter → 取消当前 Timer + 立刻重新拉一次 + 重评估是否轮询。
- 选择 polling 而非 SSE：后端 `/admin/tasks` 没有 SSE endpoint（M4.10 设计），且任务
  状态变化频率低（一次重建 ~分钟级），3s 拉一次完全够用。

#### 3. 任务详情 bottom sheet 而非新页面

- 任务详情就 progress / status / log_tail 几行，不值得专开一条路由。bottom sheet 加
  `DraggableScrollableSheet` 让用户可拽大，也能看长 log_tail。
- `SelectableText` 包 log_tail 让运维方便复制错误。
- log_tail 后端已截到最近 10 行，前端原样 render，不再二次截断。

#### 4. 文档表的 release/series 过滤

- 复用 `docs_api.list(release, series)`（M5.3 给 doc picker 用过），不引入新后端路由。
- `_docsListProvider` 用 `FutureProvider.family.autoDispose<DocListResponse, (String, String)>`
  按 (release, series) 缓存；过滤变化 = 新 family key = 自动重新拉。
- "刷新" 按钮调 `ref.invalidate(_docsListProvider)`（不带参数 = invalidate 整个 family，
  当前 active key 立刻 refetch）。

#### 5. Langfuse 外链 deep link

- 简化版：`LangfuseUrl` 走 dart-define `LANGFUSE_URL`，默认 `https://cloud.langfuse.com`。
- M5.5 不深链到 trace id —— 因为后端 admin API 不暴露最近 trace，且管理员通常进
  Langfuse 后自己按时间筛。M7 接 Langfuse Dataset 后如果有 "最近一个 trace" 入口需求
  再加。
- 用 `launchUrl(..., mode: LaunchMode.externalApplication)`：Web 走新窗口；Android 走
  系统浏览器 Intent；https scheme 在 url_launcher 默认 queries 里，无需改
  AndroidManifest。

#### 6. RebuildIndexDialog 兜底两点

- spec_id 留空 = 全量重建（后端 `IndexRebuildBody.spec_id=None` 透传给 ingestion CLI）。
- `_submitting` 锁防双击 + Error 在 dialog 内显示而不是关闭再弹 snackbar，让用户能
  直接修正 spec_id 再提交（典型场景：拼错的 `23.50`、不存在的 spec）。

#### 7. SwitchListTile.contentPadding = EdgeInsets.zero

让 force 开关与上方 TextField 左对齐；默认 SwitchListTile 左侧 16px padding 在
AlertDialog 窄宽里看着歪。

## 自主决策记录（CLAUDE.md §4.3）

1. **4 个 Tab 而非 4 条子路由**：M5.5 docs §0 写 "admin 4 路由集成测"，但路由数 ≠ URL 路由数
   —— 那里指 4 个后端 API 路由的集成覆盖。前端把 4 个功能合到一个 `/admin` 单页 + Tab，
   状态切换无需路由跳转，体验更顺。未来如果某个 Tab 内容膨胀（比如用户管理），再拆子路由。

2. **复用 `/docs` 而不是新加 `/admin/docs` 后端路由**：M5.5 docs §7 "文档表" 描述与
   `/docs` 返回完全等价，没必要新加后端路由。后端 admin.py 也没列 `/admin/docs`。

3. **任务面板轮询 3s 间隔写死**：可配置化没意义，且不能调太快（后端 PG 查询）。3s
   是与 ChatGPT 工具集 / Langfuse 等大量同类型 dashboard 一致的体感。

4. **rebuild 弹框 force 默认 false**：force=true 触发 purge，破坏性强，让管理员明示打开。

5. **Langfuse 不深链 trace_id**：M5.5 范围内 admin 没有"最近一次 trace" 的入口数据；
   管理员从 LangFuse 控制台自己 navigate 即可。M7 接 Langfuse Dataset 后再考虑。

6. **`TabController` 用 initState 初始化而非 `late final` 字段表达式**：非 admin 路径
   返回 admin_forbidden（不渲染 TabBar）时，State.dispose() 会触发 `_tab.dispose()`，
   `late final` 在 dispose 时延迟初始化会触发 `vsync: this` 在 State 已开始拆卸时报错。
   `initState` 里确定性创建消除竞态。

7. **FakeAdminApi 4 个错误注入字段（statsErr/tasksErr/rebuildErr）**：让测试一行注入
   就能验证每条 API 的 UI 错误分支，不用 mock dio adapter 重写脚本。

8. **`fake_admin_api.dart` 拿 `buildStats` / `buildTask` 工厂导出**：与 `fake_docs_api.dart`
   的 `buildDocOut` 等保持一致风格，让 widget test 写起来短。

9. **AdminDashboard 不做 keep-alive**：切换 Tab 时旧 Tab widget 不缓存（默认行为）；
   docs 列表 / stats 都用 `autoDispose` provider，离开 Tab 立刻释放；回来重新拉一次
   比缓存陈数据准确。任务面板 stateful，state 包含 polling timer，重进会重新初始化，
   也是想要的行为。

10. **`if (async case AsyncData(:final value))` 模式匹配**：用 Dart 3 sealed pattern
    matching 替代 `async.maybeWhen(data: ...)`，避免 maybeWhen orElse 分支 dead code。
    Dart 分析器认可此模式，无额外 lint。

## 剩余风险 / 已知问题

- **任务面板 3s 轮询在 task 多时浪费**：若有 > 50 个 queued/running 任务（小规模多用户
  不会触发），3s 拉 50 行偏吃带宽。生产期 M7 后再加分页或 EventSource。

- **rebuild 提交后未自动切到任务 Tab**：snackbar 提示 "去任务页查看进度"，但需用户手
  点切换。M5.6 可加自动跳转 Tab，但目前体验也能接受（snackbar 文案显式提示）。

- **docs 过滤"刷新"按钮 invalidate 整个 family**：如果用户先后试过几个 (release, series)
  组合，按"刷新"会清掉所有缓存。功能正确但偏粗放；管理员日常用不到边缘 case。

- **Langfuse 外链没有可点击 trace_id**：见自主决策 #5。M7+ 看是否要加。

- **AdminDashboard 内 RBAC 兜底文案 "没有访问权限"**：理论上不可达（router redirect 早
  拦截），写出来主要是为了 future-proof：如果开发新 admin sub-route 时忘了把它加入
  `_adminRoutes` 集合，至少页面不会崩。
