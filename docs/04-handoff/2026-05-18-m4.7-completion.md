## 2. SSE event 表落地

10 类事件按 `03-agent.md §7` 对齐：

| event | 触发源 | data 字段 |
|---|---|---|
| `run_start` | 路由入口 | `run_id` / `session_id` / `message_id` |
| `node_start` | `on_chain_start` ∩ `_NODE_NAMES` | `node` |
| `node_end` | `on_chain_end` ∩ `_NODE_NAMES` | `node` / `duration_ms` / `summary`（按 node 类型挑 query_class、rewritten_queries_count 等） |
| `chunks_hit` | retrieve 节点 `adispatch_custom_event` | `chunks[]` (chunk_id / score_dense / score_sparse / fused_score / preview) — 不含 rerank_score |
| `chunks_rerank` | rerank 节点 `adispatch_custom_event` | `chunks[]` 必含 `rerank_score`（fallback `fused_score`） |
| `token` | `on_chat_model_stream` 的 AIMessageChunk.content | `delta` |
| `final` | 顶层 `on_chain_end(name="LangGraph")` 后 | `message_id` / `answer` / `citations[]` / `confidence` |
| `end` | stream 结束（任何分支后） | `{}` |
| `cancelled` | `final_state.cancelled=True` 或 `asyncio.CancelledError` | `{"reason":"user_cancelled"}` |
| `error` | 任意 `Exception` 兜底 | `code` / `message` |

## 3. 关键决策落地点

- **Q6**：`chunks_hit` 在 `retrieve_node`（M4.3 已就位） / **Q7** `chunks_rerank` 在 `rerank_node`（本里程碑补齐）；两者**字段不同**。

- **Q14**：`archived_branch` 会话拒绝 `PATCH title`（409）、拒绝 `POST messages`（409），但允许 `DELETE`。

## 5. 自主决策记录（CLAUDE.md §4.3）

- `history_compactor` 选用 `mimo-v2.5`（非 pro）压缩，节约成本 — Q10 决策已写入文档。
- `EventSourceResponse` 选 `sse-starlette` 三方库而非手写 chunked response — fastapi/starlette 生态内，省去自己实现 ping/重连协议。
- 路由优先读 `app.state.agent_graph` 测试注入，再 fallback `app.agent.tgpp_agent` 单例 — 避免测试启动时连真依赖（Qdrant / LiteLLM）。
- `Message.status` migration 默认 `'ok'`，避免回填脚本 — 老数据天然兼容。
- `_summary_for_node_end` 节点输出按白名单字段挑选，list 字段转 count — 控制 SSE 体积（每个 node_end 一般 < 200 字节）。
- `_NODE_NAMES` 白名单过滤非节点事件（graph root / reducer），防止前端收到无意义的 on_chain_start。
