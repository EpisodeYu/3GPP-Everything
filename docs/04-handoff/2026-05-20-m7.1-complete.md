## 2. 端到端契约（runner ↔ backend）

```
load_golden(yaml)
  → for each item:
      POST /api/v1/sessions             {title, mode_default}
      POST /api/v1/sessions/{sid}/messages  {content, mode}
      ← SSE: run_start | node_start | node_end | chunks_hit | chunks_rerank
              | token (×N) | final | end | cancelled | error
      consume_sse_stream(line_iter) → AgentResponse
      compute_eval_metrics(item, resp) → EvalResult
  → results.json + report.md（eval-results/{ts}/）
```

事件名 / 字段契约硬挂在 `_apply_event(resp, ev)` 一处；backend SSE event 名 / 字段变 →
`test_runner_smoke_against_canned_backend` 自动 catch（canned graph 注入 in-process backend）。

## 5. 自主决策记录（CLAUDE.md §4.3）

1. **not_found_phrases 双侧镜像 vs 单点 SOT**：eval 不依赖 backend、backend 不依赖 eval；选**镜像 + mirror 测试**而非 sys.path 跨包 import，避免运行时 import 失败 + 隐式耦合。两侧 tuple 字面量必须一致；test_mirror_with_backend_module 失败 = 提醒同步。
2. **runner 顺序执行不并发**：M7.1 求稳；cache / 限流 / token 余量互相干扰风险大。M7.6 上 CI 看耗时再加并发。
3. **chunks 取数优先级**：chunks_rerank → chunks_hit → citations。理由：rerank 是 generate 实际输入；hit 是预 rerank 候选；citations 是最终引用（最严格但样本量小）。
4. **httpx.MockTransport 替代 respx**：respx 是第三方依赖，httpx 内置已够 — 不引入新依赖。
5. **smoke 测试用 canned graph + ASGITransport**：本地 / CI 都能跑；catch backend SSE event schema 漂移；D13 真实评测交给 M7.6 CI 用 `RUN_LIVE_EVAL=1`。
6. **backend/conftest.py 不引入**：发现 pytest_plugins 顶层限制后，改用 import-fixture 方案（`from tests.integration.conftest import app_and_state, ...`），少一份 conftest 维护。
7. **EvalResult 性能字段占位**：`ragas_*` 4 个 + `llm_calls` + `total_cost_usd` 都 None / 0，等 M7.2 / M7.4 各自 PR 填。先把数据结构契约定下来，免得后续改类签名带 ripple。
8. **重命名 _template 示例 id 为 `example-*`**：已在 M7.0 落实（避免与 hand-* 撞库）。

## 7. 剩余风险

- **runner 顺序跑 175 题 + 真 LLM 耗时**：经验单题 5-8s（retrieve + rerank + generate 同步），全集 175 题约 15-25 分钟。daily 子集 56 题约 5-8 分钟（plan 目标 < 10 分钟 ✅）。如真跑超时，M7.6 之前先评估并发。
- **`section_path` 字段格式不一致**：backend 用 `.` 拼字符串，eval `ExpectedSpec.sections` 用 tuple；`_hits_for_metrics` 已做兼容 split，单测覆盖。若 backend 未来改成 list，runner 也兼容。
- **`forbidden` 包含 zh 字符**：substring 匹配 case-insensitive 对 zh 是无操作的（zh 无大小写），不影响正确性。
- **negative 题的 `must_say_not_found_passed` 依赖词表**：词表是硬编码；真实 agent 用其他短语回答 → 误判失败。M7 期间通过实测 daily 反馈调整词表。
