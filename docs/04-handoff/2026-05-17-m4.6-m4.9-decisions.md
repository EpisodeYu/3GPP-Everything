## 二、按段落看实施影响

### 2.1 M4.6 — 鉴权与基础

- **Q2 仅 admin 创建**：路由表不实装 `POST /api/v1/auth/register`；`POST /api/v1/users` 走 `Depends(require_role("admin"))`。
- **Q3 密码 ≥ 8**：在 `app/api/v1/auth.py` / `users.py` 的请求 schema（Pydantic v2）加 `min_length=8` 校验；不接黑名单 / 复杂度库；passlib[bcrypt] 不变。
- **Q4 限流默认值**：`app/core/ratelimit.py` 三桶按文档配置；Redis key 仍 `tgpp:rl:{user_id}:{bucket}`。
- **Q5 audit 不记 chat 原文**：`app/api/v1/chat.py` 不写 `audit_logs`；仅 `users.*` / `admin.*` / `auth.*` 触发审计。
- **Q1 chunks_meta diff 脚本**：M4.6 启动前先跑。脚本归档到 `scripts/dev/diff_chunks_meta_schema.py`（或同等位置），输出对照表；通过后归档，不挂 CI。若发现差异 → 触发 §5 升级为「停下问人」。

### 2.2 M4.7 — 会话与 SSE Chat

- **Q6 / Q7 chunks 事件**：SSE 事件序列从 9 类扩到 10 类，新增 `chunks_rerank`：

  ```
  event: chunks_hit
  data: {"chunks":[{"chunk_id":"...","spec":"23.501","section":"5.6.1","preview":"..."}, ...]}   # 无 score

  event: chunks_rerank
  data: {"chunks":[{"chunk_id":"...","spec":"23.501","section":"5.6.1","rerank_score":0.87}, ...]}
  ```

  M4.7 §12 验收清单需要从「9 类 event」改为「10 类 event」。前端 `05-frontend.md` 也要同步。

- **Q8 keepalive**：`sse-starlette` 的 `EventSourceResponse(ping=15)`；Nginx 配置写到 `07-cicd-and-deployment.md §11`。

- **Q9 仅 final 落盘**：`stream_chat()` 在 final event 之后才 `UPDATE messages SET content=?, citations=?, ...`；中断 → `messages.status = 'failed'`（status 字段在 `Message` 表上新增，简单枚举：`ok` / `failed` / `cancelled`）。

- **Q10 历史压缩一步到位**：实现位置 `app/agent/utils/history_compactor.py`：
  - 最近 N=6 条 user/assistant 直接拼入 LangGraph state；
  - 更早的消息按出现顺序 batch 化交 `mimo-v2.5`（**非 pro，省成本**）做 summary，结果以 `system: <summary>` 注入 prompt 头；
  - summary 缓存在 Redis：key = `tgpp:cache:history_summary:{session_id}:{last_message_id}`，TTL 24h；
  - 触发条件：会话累计 message_count > 8 时启用 summary 流程，否则仅取最近 N 条。
  - 上述 Redis key / 模型选择 / 阈值 8 属 Agent 自主决策范围（CLAUDE.md §4.3），如人后续要调，在 PR review 阶段提即可。

### 2.3 M4.8 — Checkpoint API

- **Q11 rollback 仅当前 session**：`POST /sessions/{sid}/rollback` 只动 `sid` 本身的 messages 与 checkpoints，不递归 fork 子会话。
- **Q12 rollback 删 checkpoint**：在 PG 事务里同时 `DELETE FROM messages WHERE session_id = ? ORDER BY created_at DESC LIMIT N` + `DELETE FROM checkpoints WHERE thread_id = ? AND checkpoint_id IN (被删 messages 的 langgraph_checkpoint_id 集合)`。注意 LangGraph 自己的 checkpoint 表名以实际 schema 为准（M4.5 接入 AsyncPostgresSaver 时已创建）。
- **Q13 GC 延后 M8**：M4.8 不实装清理任务；但 `Session` 表保留 `status='paused'` 字段，前端 M5 用作展示。`docs/04-backend-api.md §11` 风险表加一条 TODO 引用本文件 Q13。
- **Q14 archived_branch**：
  - `GET /sessions` 默认按 `status` 分组返回；`status='archived_branch'` 标记。
  - `PATCH /sessions/{sid}`：若 `status='archived_branch'` → 拒绝 title 修改，但允许 `DELETE`。
  - `POST /sessions/{sid}/messages`：若 `status='archived_branch'` → 409 Conflict。

### 2.4 M4.9 — Reader / Favorites / Notes / Feedback / Tools

- **Q15 章节树**：
  - `GET /api/v1/docs/{spec_id}` 返回结构（示例）：
    ```json
    {
      "spec_id": "23.501",
      "title": "...",
      "release": "Rel-18",
      "tree": [
        {"path": "5", "title": "...", "chunk_count": 42, "children": [
          {"path": "5.6", "title": "...", "chunk_count": 18, "children": [...]}
        ]}
      ]
    }
    ```
  - 不在该端点回正文；正文走 `GET /docs/{spec_id}/sections/{path}`。
  - 实现层：从 `chunks_meta(spec_id, section_path)` GIN 索引聚合 → Redis 缓存 1h（key `tgpp:cache:reader_tree:{spec_id}:{indexed_at}`）。
- **Q16 Favorites/Notes**：`target_type` 枚举仅 `chunk` / `message`；schema 校验在 Pydantic 层。未来扩展时新增枚举值即可（DB 字段已是 text）。
- **Q17 Feedback**：
  ```python
  class FeedbackReason(str, Enum):
      hallucination = "hallucination"
      unhelpful = "unhelpful"
      incorrect = "incorrect"
      incomplete = "incomplete"
      other = "other"

  class FeedbackBody(BaseModel):
      thumb: Literal[1, -1]
      reason: FeedbackReason | None = None
      reason_text: str | None = Field(None, max_length=500)
  ```
  - `thumb=1` 时 `reason` 可为 None；`thumb=-1` 时若 `reason=other` 建议带 `reason_text`，但不强制。
