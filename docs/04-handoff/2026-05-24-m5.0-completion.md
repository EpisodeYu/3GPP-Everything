## 自主决策记录（CLAUDE.md §4.3）

1. **M5.0 暂不引 freezed / build_runner / riverpod_generator**：当前 model 只有 `Me` / `TokenPair` 两个，手写 fromJson 共 ~25 行。docs §3 把 codegen 列入 dependencies 但 §0 未把"codegen 链路跑通"作为 M5.0 门禁；推迟到 M5.2 真正堆 SSE event / chunks model 时再开 `build_runner watch`，避免现在背 codegen + linter 的运行成本。
2. **dio 拆双实例**：`dioProvider`（带 AuthInterceptor）+ `dioPublicProvider`（裸）。`refresh` / `login` / `bootstrap-admin` 走 public，避免 refresh 自身 401 触发再次 refresh 形成循环。
3. **401 重放幂等保护**：用 `RequestOptions.extra['retried']` 标志，AuthInterceptor 在 onError 看到这标志直接 `handler.next(err)`，不再二次 refresh。
4. **AuthRedirect 实现**：自写 `_AuthRefreshNotifier extends ChangeNotifier` 桥接 Riverpod → go_router `refreshListenable`，避免 GoRouter 与 Riverpod 缺联导致 redirect 不重算。
5. **登录页字符全中文硬编码**：M5.6 才进 i18n，本里程碑不写 ARB，避免拉 `flutter gen-l10n` 进 build pipeline。
6. **bootstrap 折叠面板放在登录页**：替代再做单独 admin bootstrap 页，既满足 docs §0 "Chrome smoke 跑通 bootstrap-admin / login / /auth/me" 的串行验证，也避免新增路由。
7. **测试时序**：FakeAuthController.login 内用 `Future.delayed(Duration.zero)` 模拟异步，测试用 `pumpAndSettle()` 兜底；AsyncNotifier 初次 build 微任务也需要一次 pump 才能切到 Data 态——已在 `_pumpLogin` 内加注释说明。

## 剩余风险 / 已知问题

- **Android 真机 / Windows build**：本机无 Android SDK，apk-build 在 Linux 上无法验证；按 docs §0 由人在 Windows 跑 `make apk-build API_BASE_URL=http://<dev-ip>:8002/api/v1`。
- **flutter_secure_storage on Web**：依赖 IndexedDB；某些隐私模式 / 老 Safari 可能失败。M5.0 不解决，docs §13 已记 Safari 风险。
- **AuthController 未做 token 过期前主动刷新**：当前依赖 401 触发被动 refresh。够 MVP 用；若 access token TTL 很短，后续可加定时刷新。
- **退出登录的 `logout` 调用错误被静默吞掉**：服务端 401 / 503 都只清本地 token，避免用户卡死。已在代码注释里说明。
