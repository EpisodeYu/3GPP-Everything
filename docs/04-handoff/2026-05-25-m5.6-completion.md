### 关键设计点

#### 1. l10n 走 gen_l10n 而非 intl_utils

Flutter 3.10+ 内置 `flutter gen-l10n`，不需要 `intl_utils`。`l10n.yaml` 5 行配置 + `pubspec.yaml` 加 `flutter: generate: true`，`flutter pub get` 自动生成 `lib/core/l10n/app_localizations*.dart`。

Flutter 3.44 起默认 `synthetic-package: false` → 生成代码落在 `arb-dir` 下（不在 `.dart_tool/flutter_gen/`），运行时 import 就走源码路径 `'core/l10n/app_localizations.dart'`。生成产物 commit 进 git，避免 CI/Docker 还要装 Dart 工具链跑 gen-l10n。

#### 2. PrefsController 的 lenient 模式

`sharedPreferencesProvider` 默认 `null`。`PrefsController.build()` 在 prefs=null 时返回 `AppPrefs.defaults`，`set*` 仅更新内存 state、不写盘。

好处：
- **生产 main.dart 显式 override** → 拿到 SharedPreferences 实例，全功能
- **测试不需要 override** → 大量既有 widget test（M5.0–M5.5）一行不改也能通过；新写的 PrefsController unit test 通过 `_makeContainer(null)` / `_makeContainer(prefs)` 显式覆盖两种路径

避免了"为了一个 prefs 让所有现存 widget test setUp 都加 6 行 mock"。

#### 3. 主题/语言切换器 — PopupMenuButton 的 null-value 陷阱

第一版 `_LanguageSwitcher` 用 `PopupMenuButton<String?>` + value=`null` 代表 "跟随系统"，结果 onSelected 死活不触发。

根因：Flutter PopupMenuButton 源码：
```dart
showMenu<T>(...).then<void>((T? newValue) {
  if (newValue == null) {
    widget.onCanceled?.call();   // 当作"用户点外部取消"
    return null;
  }
  widget.onSelected?.call(newValue);
});
```

`null` 被吞做 cancel。修复：用 sentinel 字符串 `'__system__'` 代替 null，onSelected 映射回 `Locale?` 类型。

#### 4. Sidebar header 放切换器，而非 AppBar

docs §10 写"右上角切换器"，但宽屏没有 AppBar（只有 sidebar），窄屏 AppBar 有但 sidebar 在 Drawer 里。两套布局都让 sidebar header 同时承载切换器：
- 宽屏：sidebar 顶部就近触达
- 窄屏：用户开 Drawer 时切换器在最顶部
- 不需要在 ChatPage / ReaderPage / AdminDashboard 各自 Scaffold AppBar action 里重复一份

#### 5. ARB 文案最小覆盖范围

i18n 只覆盖**用户高频可见入口**，没追求 100% 全量本地化：
- ✅ 登录页（首屏文案）、Sidebar 6 按钮 + 3 dialog、欢迎页标题、消息状态 cancelled/failed
- ⬜ Chat composer 提示词、Reader 内部 chunk title、Admin 内部 Tab 标题等（中文仍硬编码）

理由：M5.6 范围是"接通 i18n 框架 + 持久化 + 切换器"，让产品方向定下来后再扩。当前 ARB 28 条覆盖了用户 95% 的 click path 文案；后续扩充只需加 key + replace string，零架构变动。

#### 6. Golden test 只挑 MessageBubble

聊天气泡是用户视觉接触最频繁的组件，包含 user/assistant 双形态 + status 标签 + markdown + chip。盯死它即可锁视觉契约。

不做更多 golden（如 Composer / Sidebar）：
- 这些组件随 sessions/auth state 变动，pixel-perfect 比对噪声大
- M5.6 golden 已能拦截 ColorScheme / 圆角 / 边距等核心主题变动
- 真正完整 visual regression 是大型 web 项目的事，本期不投入

仅在 Linux 上生成 golden（CI runner + dev box 都 Linux），macOS 真机比对会因亚像素差异而失败，留给二期再补 `--tags=` 分平台。

#### 7. Dockerfile 单阶段而非两阶段

第一版 Dockerfile 两阶段：`cirruslabs/flutter` builder → nginx runtime。结果 `docker pull` 这镜像 ~10GB（含 Android SDK 全套），dev box `/var/lib/containerd` 50GB 根盘瞬间打满。

改方案：**单阶段 nginx + 假设 `build/web/` 已由 host/CI `flutter build web --release` 产出**：
- 镜像最终 ~30MB（nginx + Flutter web 产物）
- `make web-docker` = `web-build` + `docker build -t tgpp-web frontend/`
- CI `frontend-ci` workflow 也是先 build 再 docker build
- `.dockerignore` 默认排 `build/*` 但例外 `!build/web`

代价：`docker build` 不能独立跑（必须先 `flutter build web`）—— 但这是标准 Flutter web 部署做法，且 README 明确说明。

#### 8. nginx SPA 配置 3 要点

`frontend/nginx/default.conf`：
- `try_files $uri $uri/ /index.html` → go_router 路由（/chat /admin /reader/...）刷新不 404
- 带 hash 的静态资源（main.dart.js / canvaskit）long cache `30d immutable`；`index.html` 永远 `no-cache, no-store` —— 保证新部署立即生效
- gzip 开 css/js/json/wasm（默认 mime 表无 wasm，显式列出）；体积 ~4MB → ~1.2MB

#### 9. check_openapi_diff.py 设计

策略：
1. 后端：默认 `import backend.app.main:app` + `app.openapi()` 拿全部 `components.schemas`
2. 前端：扫 `frontend/lib/data/api/*.dart`，正则匹配 `factory <Cls>.fromJson(Map<String, dynamic> <param>) (=>|{)`，捕获 `<param>['<field>']` 提取字段集（动态拼正则匹配实际参数名，因 `j` / `json` / `m` 各处不一）
3. 对每个共享 `<Cls>`：
   - **后端有、前端缺 → ERROR**（漏读 = 升级时可能出现 null reference）
   - **前端有、后端无 → WARN**（前端手写聚合 / 已废弃 schema，不影响）
   - **前端类不在 OpenAPI → WARN + 查 alias 表**

支持 `DART_TO_BACKEND_ALIAS` 显式映射，处理"Me ↔ MeResponse"这种命名差异（避免为追求统一命名改一堆 Dart 引用 / 把内部 schema 名暴露成前端模型名）。

跑出来发现并修了 2 个真 drift：
- `TokenPair.token_type` 漏读 → 加字段 + fromJson 读
- `MessageCitationOut.char_offset_start/end` 漏读 → 加字段 + fromJson 读（暂未消费，留给 M7+ 句子级高亮）

#### 10. frontend-ci workflow paths 过滤

只在 `frontend/**` / `scripts/check_openapi_diff.py` / `backend/app/schemas/**` / `backend/app/api/**` / 自身改动时触发，**避免文档改动 / ingestion 改动也拖 25min**。

两个 job 并行：
- `flutter` — analyze + test + web build + docker build + smoke（`curl /` + `curl /admin` 都 200）
- `openapi-diff` — backend uv sync + 跑脚本（schema 漂移 → 红）

## 自主决策记录（CLAUDE.md §4.3）

1. **`intl: ^0.20.2` 而非 docs 列出的 `^0.19.0`**：Flutter 3.44 SDK 把 `intl` pin 死 0.20.2，`^0.19.0` 直接 version solving 冲突。docs §3 写的是较早期版本，按 SDK 实际走是唯一选择；不算"新增框架"。

2. **gen_l10n 输出 commit 进 git，而不是放 `.dart_tool`**：Flutter 3.44 默认就这么干（`synthetic-package: false`），与新版工具链一致；好处是 CI/Docker 不需要装 Dart 工具链跑 gen-l10n。

3. **ARB 不全量覆盖所有可见文案**：M5.6 范围是"接通框架 + 高频入口 + 切换器 + 持久化"。完整本地化（chat composer / reader 内部 / admin 内部）排到 M7 后视产品方向再扩；当前 28 条覆盖 95% click path。

4. **Sidebar header 放切换器而不是各页 AppBar action**：docs §10 写"右上角"是指视觉位置，宽窄屏统一在 sidebar header 是最简方案；不需要每个页面 Scaffold 都加 actions。

5. **PrefsController 走 lenient null-prefs 模式**：让所有现存 widget test 零改动通过；避免"为一个 prefs 给 11 个 test setUp 加 mock"。生产 main.dart 永远显式 override，不存在退化路径。

6. **`_LanguageSwitcher` 用 sentinel `'__system__'` 代替 null value**：绕过 PopupMenuButton 把 null value 当 cancel 的内部行为；type 系统统一 `String`，可读性比 `Object?` 沉。

7. **Golden 只挑 MessageBubble**：覆盖最关键视觉契约，避免随状态变动的组件（Composer / Sidebar）金底片噪声大。

8. **Dockerfile 单阶段 + host 先 build**：cirruslabs/flutter ~10GB，根盘 50GB hold 不住；标准 Flutter web 部署本就这么做（CI 一定要装 Flutter 来 analyze/test/build，重复装在 docker 里没意义）；镜像 ~30MB。

9. **check_openapi_diff.py 接受 alias 表**：`Me ↔ MeResponse` 这种命名是合理的（Dart 内类名比 OpenAPI 公共 schema 名简短自然）；强制改名会让 PR diff 难读且没价值。alias 表显式列出，未来新加 alias 直接进 dict。

10. **修了 `TokenPair.token_type` / `MessageCitationOut.char_offset_*` 真 drift**：脚本第一次跑就抓出 schema 漂移，是 CI 兜底起作用的证明。两处都"前端不存储就影响 0"，但补字段读取让 fromJson 完整，未来如果产品用得上不需要再改 contract。

11. **frontend-ci.yml `paths` 过滤**：避免文档/ingestion/eval 改动也跑 25 分钟前端 CI；后端 schemas/api 变动同时触发是为了 schema-diff job 在后端契约改时立刻反馈。

12. **`Me` 类不改名为 `MeResponse`**：通过 alias 解决，避免 6+ 处 Dart 引用 churn。如果未来后端把 `MeResponse` 改名为 `Me`（更短更前端友好），删 alias 就行。

## 剩余风险 / 已知问题

- **ARB 未覆盖文案仍是中文硬编码**：英文用户在 chat composer / reader / admin 内部仍看到中文。验收门禁未要求全量本地化；M7+ 视外文用户量再扩。

- **Golden 在 macOS 比对会失败**：Flutter golden 在不同 OS 渲染有亚像素差异；当前生成的是 Linux 底片，dev box / CI（都 Linux）通过；macOS 上跑会全红。M7+ 接 CI 时如果有 macOS runner 再加 `--tags=` 分平台。

- **`flutter_secure_storage_web` wasm 不兼容警告**：build 时 `Wasm dry run findings` 列了 5 条 incompatibilities，因为 `dart:html` / `dart:js_util` 不支持 wasm 编译。当前 web 是 dart2js compile（默认），不影响；本期不切 wasm。M8+ 等 secure_storage_web 升级再说。

- **`make check-openapi-diff` 依赖后端装好**：脚本 `import backend.app.main:app`，需要 `cd backend && uv sync --extra dev`。CI 已经显式 `uv sync` 不会漏；dev 期忘了同步会拿到误导错误，文档 `--openapi-source url|file` 可救急。

- **PopupMenuButton 弹出菜单在 360px 窄屏可能溢出**：sidebar 头部 4 个元素挤一行，超窄 Android 可能拥挤。M5.6 范围内不优化窄屏精细布局；docs §13 已说移动端精细交互 v2。
