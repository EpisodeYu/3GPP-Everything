## 一、剩余风险（Risks）

### 1.1 数据/索引侧

- **R1 · M6 dense-only baseline 显著低于 M3 17-spec baseline**
  spec R@10=0.580 / section R@10=0.437 / MRR=0.236（vs M3 -23/-21pp）。文档已声明这是"草垛扩 75× 的预期稀释，不视作回归"，但 M4 接 rerank 后若 ablation 提升不足，意味着真正的检索质量风险被掩盖。
  来源：`00-overview.md §3` / `eval-results/m6-retrieval-baseline.md`

- **R2 · 全量索引磁盘逼近门槛**
  `/data` 全量后 Qdrant + BM25 + chunks 占用接近 30GB 软门槛。M6 之后若再做维度切换或双轨重建，磁盘会立刻爆。
  来源：`00-overview.md §2 磁盘门槛`

- **R3 · Chunker section_splitter 重复 chunk_id（P1）由 dedupe 兜底**
  根因未除，靠 indexer 入库 dedupe 防重。一旦 chunker 改逻辑漏 dedupe，索引会出现幽灵 chunk。
  来源：`2026-05-15-m1-poc-38331.md §6.1`

- **R4 · Table separator 兜底覆盖率仅 ~93%**
  7% 表格缺 separator 导致前端 markdown 渲染失败（已 fix 但未补 unit test 防回归）。
  来源：`2026-05-15-m1-poc-38331.md §6.2`

- **R5 · PG `chunks_meta` schema 由 ingestion 侧维护是过渡方案**
  M4.0 alembic 初版接管时，类型/索引/外键若与现状不完全一致，迁移会痛。需在 M4.0 完成报告里点名核对。
  来源：`2026-05-15-m1-indexer.md §8.1`

- **R6 · Voyage 海外延迟波动 + 单 provider 依赖**
  虽配 tenacity + 30s timeout + Jina fallback，但 fallback 路径未在 CI 跑过端到端。
  来源：`03-agent.md §13`

- **R7 · Vision 缓存命中率依赖 Redis 永久缓存**
  Redis 若被清空或过期策略误配，全量 Vision 调用费用会翻倍。
  来源：`02-ingestion-and-indexing.md §7`

### 1.2 Agent/生成侧

- **R8 · LLM 引用幻觉无自动兜底**
  self-RAG 只判 verdict，不真正核对 citation 是否真实存在于 retrieved chunks。M7 nightly eval 才能量化。
  来源：`03-agent.md §13`

- **R9 · self-RAG 强制收敛在低质答案**
  `retry_count >= 2` 强制 accept，意味着差答案直接出。未来需要前端能区分"高/中/低 verdict"给用户标注。
  来源：`03-agent.md §M4.3 / §4.8`

- **R10 · 工具节点权限滥用**
  `explicit_tools` 严格限制 LLM 自主调 web_search 等，但 prompt 中边界仍可能被模型误判（如把"问 5G 概念"判成需要 web 检索）。混淆矩阵未跑。
  来源：`03-agent.md §13`

- **R11 · M4.2 spec hit 真实环境 2/5**
  当前 simple 路径检索质量弱，仅由 M7 nightly 严格评测保障，M4 不作为门禁。如果 M7 暴露问题，需要回头调 retrieval / rerank / prompt。
  来源：`03-agent.md §M4.2`

- **R12 · AsyncPostgresSaver 在 LangGraph v3 的兼容性**
  M4.5 自主决策接 `AsyncPostgresSaver`，但 `NodeInterrupt` 在 v2 之后可能迁到 `langgraph.types.interrupt`。M5 之前必须监控版本变化。
  来源：`03-agent.md §M4.5 自主决策记录`

### 1.3 工程/运维侧

- **R13 · LiteLLM proxy 单点故障**
  无 HA。proxy 挂了所有 LLM/Embedding/rerank 全断。
  来源：`04-backend-api.md §11`

- **R14 · paused/失败 run 的 checkpoint 堆积**
  无后台 GC，PG 会持续膨胀。M4.5 完成 saver 接入但没接清理任务。
  来源：`03-agent.md §13` / `04-backend-api.md §11`

- **R15 · SSE 在 Nginx 后被缓冲**
  生产 Nginx 需 `proxy_buffering off` + keepalive ping。当前 deploy 配置未验证。
  来源：`04-backend-api.md §11` / `07-cicd-and-deployment.md §11`

- **R16 · 2 核服务器 worker=3 临近上限**
  M6 实测可承受，但叠加在线 agent 请求时 load 可能 > 5。需要在线流量基线后再调。
  来源：`2026-05-16-m1-to-m2-decisions.md §5`

- **R17 · 多轮对话 context 撑爆**
  当前未做历史压缩，长会话直接打满模型 context。
  来源：`04-backend-api.md §11`
