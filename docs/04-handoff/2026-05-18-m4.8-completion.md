## 3. 自主决策记录（CLAUDE.md §4.3）

1. **`_has_inflight_run` 复用 M4.7 stub 模式**：用 `Message.role='assistant' AND status='ok' AND content=''` 作为 inflight 信号，而非新加 `status='running'` 状态。理由：M4.7 已为 SSE final/cancelled 路径专门做了 stub→update 协议，复用避免引入新状态字段。
2. **`fork` 拒绝 `archived_branch` 源会话（409）**：而不是允许从 archived 上再 fork。理由：fork 的语义是"从活动会话的某 checkpoint 起新分支"，对 archived（已是某次 fork 的源）再 fork 会让 `forked_from_session_id` 链失去含义；如有需求让人 explicit 解 archive 再操作。
3. **`rollback` 删 message 走 `delete + MessageCitation.in_(ids_to_delete)` 双步**：而不是依赖 FK ON DELETE CASCADE（PG 侧已有）。理由：保兼容性 + 测试用 SQLite ON DELETE 行为不一致；双步显式更稳。
4. **`resume` 状态机决策**：必须 `status='paused'` 才允许 resume（否则 409 `session_not_paused`）；resume 后置 `status='active'`。不引入"resuming"中间态。
5. **SSE 续跑流程复用**：`chat._build_sse_stream` 从原 `send_message` 内部抽出为模块级 helper，签名 `*, graph, sid, assistant_msg_id, run_id, initial_state, db`。`initial_state=None` 走 LangGraph 续跑；非 None 走 M4.7 原本的"喂初始 state"路径。两条路径共用 SSE 事件映射与 DB final 落库逻辑。
6. **`interrupt({"reason": "..."})` payload 选 dict 而非 str**：与 LangGraph types 文档示例一致；测试断言用 `str(i.value)` 子串匹配（"cancelled" / "paused"），保留向前兼容性。
7. **A.4 subset 测试阈值从 0.6 调到 0.65**：与 todo D13 决策口径完全一致（context recall ≥ 0.65）。M7 eval runner 实装后立即生效。

## 5. 剩余风险

| 风险 | 估计影响 | 缓解 |
|---|---|---|
| `resume` 在 PostgresSaver 接入前是 best-effort（无 checkpoint 时直接清 paused flag 续跑） | M4.10 端到端真跑可能暴露状态不一致 | resume 路由对 graph.aupdate_state 失败做了 `contextlib.suppress`；待 M4.10 接 PostgresSaver 后由真 thread snapshot 兜底 |
| `_has_inflight_run` 复用 M4.7 stub 模式（content='' AND status='ok'）：若 M4.7 stub 协议改字段会同步坏 | 跨里程碑耦合 | 集中在 `app/api/v1/checkpoint.py:_has_inflight_run` 一处；如 M4.10 改协议同步改这里即可（检索成本低）|
| `interrupt()` payload schema 未约束（值是 dict 但 key 是约定） | 跨节点的 paused/cancelled 检查依赖 `str(value)` 包含特定子串 | 集成测 4 个 case 已覆盖；如未来需要更强 schema，集中在 `app/agent/nodes/*` 9 处替换即可 |
| ingestion A.1 兜底测覆盖了"缺 separator → 降级 paragraph"行为，但**没有覆盖"真伪 separator 边界 case"**（如 `|--|--|` 仅 2 dash 是否合规）| 极小概率回归 | 已有 `test_table_delim_regex_rejects_empty_cell_row` 覆盖；后续若发现新 GSMA marker 形态再补 |
