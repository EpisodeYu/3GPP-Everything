## 自主决策记录（CLAUDE.md §4.3）

1. **不引 freezed / build_runner**：会话 model 仍是手写 fromJson（约 30 行）。继承 M5.0 决策——把 codegen 留到 M5.2 真用到 SSE event union / chunks model 时一次开起来，避免现在背 build_runner 噪声。
2. **会话列表用单一 `AsyncNotifier<List<SessionOut>>`**：不拆 "current session" 单独 provider，sidebar + chat page 都从同一份读，避免双源同步；create / delete 走**乐观更新 + 失败回滚**而非全量 refresh，避免 sidebar 闪烁。
3. **响应式断点 840px**：来自 docs §4 原文 "宽屏（>=840px）侧栏固定"；用 `LayoutBuilder` 测真实 constraint（而非 `MediaQuery`），不被外部 BoxConstraints 误导。
4. **窄屏继续走 Drawer 而非底部 BottomSheet**：Material 标准；docs §4 写明 "窄屏侧栏抽屉化"。
5. **chat_page 改成 `ChatPage({String? sessionId})` 而非 `/chat` 和 `/sessions/:sid` 各一个 widget**：核心渲染分支只有 3 种（null / 命中 / miss），共用一个 widget 比拆两个更省代码；M5.2 接 SSE 时也只在 "命中" 分支里加 ChatRunController，不会污染 welcome / miss。
6. **删除走 AlertDialog 二次确认**：会话删除不可逆（后端 `ondelete=CASCADE` 一刀切 messages + citations），不做撤销；二次确认 + 红色按钮已经是常规做法。
7. **重命名用 dialog 输入而非 inline 编辑**：inline 编辑要做 long-press / tap-to-edit 状态切换 + IME 焦点管理 + ESC 撤销，复杂度不值得；dialog 单刀直入。
8. **`SessionsApi` 默认拉前 200 条**：MVP demo 期一个用户不会有上百会话；翻页等 M5.4 / M5.6 真有需要再做（docs §0 也没把翻页列入 M5.1 门禁）。
9. **`integration_test/login_flow_test.dart` 同步加 `sessionsApiProvider.overrideWithValue(_EmptySessionsApi())`**：M5.1 让 AppShell mount 时会拉一次 sessions，smoke 测试在浏览器跑就会打真后端；用空 API 桩避开。
10. **`PopupMenuButton` 不加 "复制 / 收藏 / thumb / 笔记"**：那些是 docs §5.4 / §5.6 assistant 消息长按菜单的内容，不在 session tile 菜单里；M5.4 才接。session tile 菜单只放 "重命名 / 删除" 两项，最小集。
11. **`SessionsApi.list` 查询字段使用 `page` / `page_size`**：与后端 `list_sessions` 的 `Query(default=1, ge=1)` / `Query(default=50, ge=1, le=200)` 完全对齐。
12. **新增 `frontend/scripts/chrome-headless-wrapper.sh` + Makefile `web-smoke-chrome-headless` target**：本机 Linux 服务器无 X / 无 sudo（装不了 Xvfb），但 google-chrome 147 已装、chromedriver 147 已装。wrapper 通过 `CHROME_EXECUTABLE` 把 chrome 启动参数固定为 `--headless=new --no-sandbox --disable-gpu --disable-dev-shm-usage --window-size=1280,800`，让 `flutter drive -d chrome` 不再依赖 X 显示，也避免 < 840px 窗口触发 AppShell drawer 分支（M5.1 integration_test 断言 sidebar 文案）。这条通路与已有 `web-smoke` (`-d web-server`) 形成双保险，CI / 本机 dev 都能用。

## 剩余风险 / 已知问题

- **真"有窗口"chrome 仍不在本机覆盖范围**：本机 Linux 无 X server 且无 sudo（装不了 Xvfb），所有 chrome 跑测试都走 `--headless=new`。`make web-smoke` (`-d web-server`) 与新增的 `make web-smoke-chrome-headless` (`-d chrome` + wrapper) 在两条通路上都跑通真 chrome 引擎，是 CI 等价物；但 docs §14 的 `[human]` 仍需要人在 macOS/Windows 真带窗口浏览器上手动验证一遍 UX（特别是 sidebar / drawer 切换的动效与点击响应）。
- **archived_branch session 删除是否需要"先解绑 forked_from_session_id"**：M5.1 只是按 status 分组；后端 `DELETE /sessions/{sid}` 在 cascade 下会一刀切，但如果 fork 链很长 UX 上需要进一步引导，留给 M5.4 fork/rollback UX 段处理。
- **窄屏 drawer 在某些 viewport 切换时（拉宽 → 拉窄 → 拉宽）可能短暂出现 sidebar + drawer 双绘**：LayoutBuilder + Scaffold 的常规边界，没看到实际 jank；如果 M5.6 golden test 抓到再细调。
- **`AsyncNotifier.refresh()` 当前只在 sidebar 错误状态 "重试" 按钮调用**：会话列表与服务端可能出现长期 drift（手机断网 / 别端登录新增）；MVP 阶段不做 polling 或 WebSocket；M5.5 admin 可能加手动刷新。
- **重命名 dialog 关闭路径没做 controller dispose**：Flutter 这种 short-lived dialog controller 不显式 dispose 也不漏，但 lint 严格时可能报；本次未触发，留意。
